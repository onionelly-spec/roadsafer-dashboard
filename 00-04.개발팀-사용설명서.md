## 1. 프로젝트 개요 및 기술 스택

본 퍼블리싱 산출물은 **로드세이프티시스템(RSS)** 서비스의 사용자 사이트와 관리자(adm) 사이트 전체 화면을 포함합니다. 특정 프론트엔드 프레임워크(React, Vue 등)에 종속되지 않은 순수 HTML/CSS/JavaScript로 작성되었으며, Java + MariaDB 기반 Spring Boot + Thymeleaf 환경으로의 전환을 전제로 설계되었습니다.

| 항목 | 사용 기술 | 비고 |
|------|----------|------|
| 마크업 | HTML5 시맨틱 태그 | `<main>`, `<section>`, `<article>`, `<nav>` 등 |
| CSS | Tailwind CSS (CDN) + 커스텀 CSS | `rss-common.css`, `adm/adm-common-v2.css` |
| JavaScript | Vanilla JS (ES6+) | `rss-common.js`, `adm/adm-common.js` |
| 아이콘 | Lucide Icons (CDN) | `data-lucide` 속성 방식 |
| 폰트 | Pretendard Variable (CDN) | `@import` 방식 |
| 반응형 | 모바일 · 태블릿 · 데스크탑 | Tailwind 브레이크포인트 기준 |

> **Tailwind CSS CDN 주의사항**: 현재 개발 편의를 위해 CDN 방식을 사용합니다. 운영 배포 시에는 Tailwind CLI 또는 PostCSS를 이용해 **빌드된 정적 CSS 파일**로 교체하는 것을 권장합니다.

## 4. 공통 컴포넌트 적용 방법

### 4-1. 현재 퍼블리싱 방식 (fetch 삽입)

현재 퍼블리싱 파일은 `rss-common.js` / `adm-common.js`의 `fetch()` API를 이용해 사이드바·헤더·푸터를 각 페이지에 동적으로 삽입합니다.

**사용자 사이트 마운트 포인트:**

```html
<div id="rss-sidebar-wrap"></div>
<div id="rss-header-wrap" data-title="페이지 타이틀"></div>
<!-- 본문 내용 -->
<div id="rss-footer-wrap"></div>
```

**관리자 사이트 마운트 포인트:**

```html
<div id="adm-topbar-wrap"></div>
<div id="adm-sidebar-wrap"></div>
<!-- 본문 내용 -->
```

> **로컬 테스트 주의사항**: `file://` 프로토콜에서는 CORS 정책으로 `fetch()`가 동작하지 않아 사이드바·헤더가 표시되지 않습니다. 반드시 **VS Code Live Server** 또는 로컬 웹 서버 환경에서 확인하십시오.

### 4-2. Thymeleaf Fragment 전환 방법

Spring Boot + Thymeleaf 환경으로 전환 시 아래 절차를 따릅니다.

**Step 1.** `sidebar.html`, `header.html`, `footer.html`(사용자) 또는 `adm-sidebar.html`, `adm-topbar.html`(관리자)의 내용을 `/templates/fragments/` 폴더로 이동합니다.

**Step 2.** `rss-common.js` 상단의 `0. 공통 컴포넌트 fetch 삽입` 블록 전체를 **삭제 또는 주석 처리**합니다. `adm-common.js`의 `1. 공통 컴포넌트 삽입` 블록도 동일하게 처리합니다.

**Step 3.** 각 HTML 페이지의 마운트 포인트를 Thymeleaf 문법으로 교체합니다.

```html
<!-- 변경 전 (퍼블리싱) -->
<div id="rss-sidebar-wrap"></div>
<div id="rss-header-wrap" data-title="공지사항"></div>

<!-- 변경 후 (Thymeleaf) -->
<div th:replace="~{fragments/sidebar :: sidebar}"></div>
<div th:replace="~{fragments/header :: header(title='공지사항')}"></div>
```

### 4-3. 활성 메뉴 (Active) 처리

현재 `rss-common.js`와 `adm-common.js`는 현재 URL을 파싱하여 사이드바 메뉴에 `active` 클래스를 자동으로 부여합니다. Thymeleaf 전환 시 컨트롤러에서 현재 메뉴 코드를 모델에 담아 전달하고 조건문으로 처리하는 방식을 권장합니다.

```html
<!-- sidebar.html Fragment 내부 예시 -->
<a th:href="@{/notice}"
   class="rss-nav__link"
   th:classappend="${currentMenu == 'notice'} ? ' active'">
  <span class="rss-nav__icon"><i data-lucide="bell"></i></span>
  <span class="rss-nav__label">공지사항</span>
</a>
```

## 5. 데이터 바인딩 가이드

### 5-1. 주석 규칙

퍼블리싱 소스 코드 내에는 아래 형식의 HTML 주석으로 데이터 바인딩 위치를 표시했습니다.

| 주석 형식 | 의미 |
|----------|------|
| `<!-- [DB] tbl_construction.order_corp_nm -->` | 연관 DB 테이블 및 컬럼 정보 |
| `<!-- [Java] th:text="${construction.orderCorpNm}" -->` | 권장 Thymeleaf 표현식 |
| `<!-- [Java] th:each="item : ${list}" -->` | 반복 처리 위치 |
| `<!-- [Java] th:href="@{/adm/order-corp/write/{seq}(seq=${item.seq})}" -->` | 동적 링크 |
| `<!-- [Java] th:selected="${searchType == 'orderCorp'}" -->` | 선택 상태 바인딩 |

### 5-2. 더미 데이터 교체 방법

목록 테이블에는 디자인 확인용 더미 행이 반복적으로 하드코딩되어 있습니다. 아래 패턴으로 교체하십시오.

```html
<!-- 변경 전 (퍼블리싱 더미) -->
<tbody id="constructionListBody">
  <tr>
    <td>한국개발(주)</td>  <!-- [Java] th:text="${item.orderCorpNm}" -->
    <td>호남선 순천방향 포장공사</td>  <!-- [Java] th:text="${item.constructionNm}" -->
  </tr>
  <tr><!-- 더미 반복 행 --></tr>
</tbody>

<!-- 변경 후 (Thymeleaf) -->
<tbody id="constructionListBody">
  <tr th:each="item : ${constructionList}">
    <td th:text="${item.orderCorpNm}"></td>
    <td th:text="${item.constructionNm}"></td>
  </tr>
</tbody>
```

### 5-3. 폼 요소 name/id 규칙

`input`, `select`, `textarea`의 `name`과 `id` 속성은 **DB 컬럼명의 camelCase 변환**을 기준으로 작성되었습니다. 용어사전 `00-03.용어사전.md`를 참조하여 DTO(VO) 필드명과 매핑하십시오.

```html
<!-- 예시: 발주기업 등록 폼 -->
<input type="text" id="orderCorpNm" name="orderCorpNm" />
<!-- DB: order_corp_nm, DTO: orderCorpNm -->

<input type="text" id="bizNo1" name="bizNo1" />
<!-- DB: biz_no (분할 입력 1/3), DTO: bizNo1 -->
```

### 5-4. 사용자 역할 분기 처리

사용자 사이트에서 역할(기업주/차단직원)에 따라 메뉴와 UI 요소가 분기됩니다. 현재 퍼블리싱에서는 `data-role-only` 속성으로 표시하고 있으며, Thymeleaf 전환 시 세션의 역할 값으로 조건 처리하십시오.

```html
<!-- 퍼블리싱: data-role-only 속성 -->
<li data-role-only="owner">TMA 카드 관리</li>

<!-- Thymeleaf 전환 예시 -->
<li th:if="${session.loginUser.role == 'owner'}">TMA 카드 관리</li>
```

**정보변경 페이지 분기**: 기업주는 `profile.html`, 차단직원은 `profile-staff.html`로 분기됩니다.
